# Documentation Guide for `systemfehler`

This folder contains all public-facing documentation, including:

- Political manifestos and mission statements
- Technical documentation for developers
- Guides for maintainers, contributors, and users
- Metadata schema, API structure, and internal architecture

## Structure Suggestions

- `intro.md`: Project vision, background, and goals
- `architecture.md`: Overview of code and data flow
- `api.md`: Explanation of JSON structure and endpoints
- `crawlers.md`: How data is extracted, normalized, and updated
- `frontend.md`: Nuxt 3 structure, accessibility layers, i18n
- `contributing.md`: Guidelines adapted for doc writers

## Style Tips

- Use clear, inclusive language
- Use headings (`##`) liberally
- Prefer passive formulations in formal or shared documents
- Link to source code where relevant (e.g. `/apps/fe/components/...`)
- Keep all docs under version control – no Google Docs :)

> ✊ Write as if you're teaching the next movement.
