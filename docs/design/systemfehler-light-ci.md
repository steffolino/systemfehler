# Systemfehler Light CI

This project uses a light, calm, trustworthy visual language.

## Core Principles

- Simplicity first.
- Clear hierarchy over decoration.
- Friendly, direct, human tone.
- Helpful guide, not a corporate campaign.
- Decorative graphics only when they support orientation.

## Brand Assets

Use the provided graphics intentionally:

- `BrandIcon` for compact surfaces, footer chips, and small brand marks.
- `BrandWordmark` for the header and other primary brand moments.
- `BrandHeroGraphic` for the top hero area on the search page.

Use cases that fit:

- header branding
- hero sections
- OG-style hero visuals
- compact footer or legal navigation markers

Avoid using graphics just to fill space.

## Do Not

- do not place illustrations into every card
- do not add decorative art to forms, tables, or result lists
- do not replace readable text with text baked into images
- do not stack too many brand treatments in one view

## Layout Rules

- mobile first
- one clear primary action per section
- spacious but not empty
- left-aligned content
- keep long text areas clean

## Tone Rules

- concise
- concrete
- no marketing language
- no bureaucratic phrasing
- no over-promising

## Implementation Rule

When adding a new screen or section, ask:

1. Does this image help the user understand where they are?
2. Is the asset stronger than plain text here?
3. Does it add clarity rather than noise?

If not, leave it out.
